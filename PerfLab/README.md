# PerfLab — аллокации под микроскопом + нагрузочный тест

Pet-проект №11–12 из плана. Закрывает раздел 14 из «200 ответов»: **Производительность и highload**.

## Две части

### 1. BenchmarkDotNet: убрать аллокации с горячего пути

```bash
dotnet run --project src/PerfLab.Benchmarks -c Release -- --job short
```

Результаты с этой машины (1000 строк реестра / 500 строк отчёта):

| Метод | Время | Аллокации | |
|---|---:|---:|---|
| `Split` + `Parse` | 77.7 µs | 168 КБ | baseline |
| `Span` + `Parse(span)` | 48.4 µs | **0 Б** | срезы смотрят в исходную строку |
| Конкатенация в цикле | 318 µs | **6.2 МБ** | каждая `+=` копирует всё накопленное |
| `StringBuilder` (с capacity) | 16.2 µs | 54 КБ | ×20 быстрее, ×114 меньше мусора |

Почему аллокации важнее наносекунд: мусор в Gen0 → чаще сборки → паузы у **всех**
запросов процесса. `[MemoryDiagnoser]` — обязательная колонка любого бенчмарка.

### 2. NBomber: p95/p99 от RPS + thread pool starvation вживую

```bash
dotnet run --project src/PerfLab.LoadTest -c Release   # ~1 мин, самодостаточен
```

Два эндпоинта «ждут БД» 50 мс: `/async` — `await Task.Delay`, `/blocking` — `Thread.Sleep`
(модель sync-over-async в проде). Результаты:

| endpoint | RPS | p50 | p95 | p99 |
|---|---:|---:|---:|---:|
| /async | 50→400 | ~62 мс | ~64 мс | ~66 мс — **плоско** |
| /blocking | 50 | 56 мс | 65 мс | 67 мс |
| /blocking | 200 | 59 мс | 90 мс | 105 мс |
| /blocking | 400 | 129 мс | 317 мс | **476 мс** |

Блокирующему эндпоинту нужно `rate × 0.05` потоков одновременно; пул доращивает потоки
медленно → очередь пула растёт → хвост латентности взрывается. Async не занимает поток
на время ожидания — кривая плоская. **Один и тот же «50 мс работы» — семикратная разница
в p99 уже на 400 RPS.**

## Мостик к собесу

Корректные замеры (Release, warmup, медиана vs среднее) · чтение p50/p95/p99 и почему
среднее врёт · поиск bottleneck: CPU vs I/O vs пул потоков · Span/Memory и stackalloc ·
поколения GC и цена Gen0-мусора · thread pool starvation: симптомы (latency растёт при
низком CPU!) и диагностика (`dotnet-counters`, ThreadPool Queue Length).
