# SystemDesign — 5 письменных разборов

Pet-проект №4 из чек-листа (письменный, не код). Закрывает раздел 16 из «200 ответов».

## Шаблон (им же пользуйся на собесе, вслух, по шагам)

1. **Требования** — функциональные + нефункциональные (SLA, латентность, консистентность). Задай уточняющие вопросы — это оценивается.
2. **Оценка нагрузки** — RPS, объём данных, рост. Порядки, не точность.
3. **Схема** — компоненты и стрелки. Начни с простой, усложняй по запросу.
4. **Детализация** — данные, API, ключевые потоки. Здесь всплывают паттерны из твоих проектов.
5. **Bottleneck и эволюция** — что упадёт первым при ×10 и что с этим делать.
